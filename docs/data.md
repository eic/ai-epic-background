# Data

Available **background-mixed** datasets in Rucio. Source of truth:

```bash
rucio did list --filter 'is_background_mixed=true' 'epic:*'
```

This page is regenerated by the `update-data-md` skill — see
[`.claude/skills/update-data-md/SKILL.md`](https://github.com/eic/ai-epic-background/blob/main/.claude/skills/update-data-md/SKILL.md)
for the procedure, and `scripts/update_data_md.py` for the parser.

_Last updated: **2026-05-26**_

## Conventions

- **FULL** — DD4hep simulation output (sim-level, full Geant4 hits).
- **RECO** — EICrecon reconstruction output (reco-level).
- Each table covers one `(campaign, data_name)` pair. Rows are sorted
  by beam energy, then by `minQ2`, with the **FULL** row above its
  **RECO** twin.
- `main/` branch / CI / Test entries are filtered out — only versioned
  campaigns are shown (21 `main/` entries skipped on the
  last run).
- `epic_craterlake` is the only detector configuration; the full DID
  including that segment is shown in each row.
- Use the per-row 📋 to copy a single DID, or the top-of-table button
  to copy all DIDs in a dataset (newline-separated, ready to paste
  into `rucio download`, a shell loop, or a config file).
